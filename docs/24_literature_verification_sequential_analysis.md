# 24 Literature Verification: Sequential Analysis and Stopping Rules

This document continues Stage 2 of the five-stage refinement roadmap.

It checks sequential analysis, optimal stopping, and stopping-rule literature against UDAM claims about repeated observation and checking loops after anchor loss.

The goal is not to prove UDAM from sequential analysis.

The goal is to classify which UDAM components are directly supported, partially supported, analogy-only, unsupported, or in need of wording correction.

## Sources checked

Reference candidates:

- Abraham Wald, "Sequential Tests of Statistical Hypotheses," Annals of Mathematical Statistics, 1945.
- Abraham Wald and Jacob Wolfowitz, "Optimum Character of the Sequential Probability Ratio Test," Annals of Mathematical Statistics, 1948.
- K. J. Arrow, David Blackwell, and M. A. Girshick, "Bayes and Minimax Solutions of Sequential Decision Problems," Econometrica, 1949.
- Y. S. Chow, Herbert Robbins, and David Siegmund, *Great Expectations: The Theory of Optimal Stopping*, 1971.
- David Siegmund, *Sequential Analysis: Tests and Confidence Intervals*, 1985.
- Christopher Jennison and Bruce Turnbull, *Group Sequential Methods with Applications to Clinical Trials*, 1999.
- Chen Cheng and Xun Huan, "Optimal Stopping for Sequential Bayesian Experimental Design," arXiv:2509.21734, 2025.

Core literature idea:

```text
data are evaluated as they arrive, and the decision to continue sampling or stop is governed by a stopping rule
```

Optimal-stopping version:

```text
continue only when expected continuation value exceeds stopping value plus sampling or observation cost
```

## Minimal notation

| Symbol | Meaning in this note | Unit | Definition | Domain / assumption | Type |
|---|---|---:|---|---|---|
| `i` | observation index | count | current repeated-observation step | integer `i >= 1` | scalar |
| `y_i` | result of observation `i` | dimensionless | observed signal at step `i` | generated by an observation process | random variable |
| `H_i` | observation history | dimensionless | `H_i = (y_1, ..., y_i)` | available after `i` observations | sequence |
| `S` | UDAM-relevant state | dimensionless | state being inferred after anchor loss | discrete or continuous | random variable |
| `p(S | H_i)` | posterior after `i` observations | probability | belief over `S` after history `H_i` | sums or integrates to 1 | distribution |
| `a_i` | action after `i` observations | dimensionless | action selected using `H_i` | belongs to feasible action set | decision variable |
| `tau_stop` | stopping time | count | index at which observation stops | adapted to observed history | random variable |
| `C_obs,i` | cost of next observation | utility unit | cost of acquiring or processing observation `i+1` | usually non-negative | scalar |
| `V_stop(i)` | value of stopping at step `i` | utility unit | expected value of acting now using `H_i` | criterion-dependent | scalar |
| `V_cont(i)` | value of continuing from step `i` | utility unit | expected value after at least one more observation, including later choices | criterion-dependent | scalar |
| `MOV_i` | UDAM marginal observation value | utility unit | expected net value of another observation under UDAM | positive means continue is locally favorable | scalar |

Dimension check:

```text
V_cont(i) - V_stop(i) - C_obs,i = utility unit
```

Therefore a marginal observation rule is dimensionally valid only if observation costs and action values are expressed in the same utility unit.

## Literature pattern

Sequential analysis differs from fixed-sample analysis:

```text
fixed-sample analysis: choose sample size first -> collect all data -> decide
sequential analysis: observe -> evaluate -> continue or stop -> decide
```

A minimal stopping comparison is:

```text
continue if V_cont(i) > V_stop(i)
stop if V_cont(i) <= V_stop(i)
```

If the cost of the next observation is separated:

```text
continue if expected gain from another observation > observation cost
```

This is close to the UDAM rule:

```text
MOV_i > 0
```

But this match is not exact unless `MOV_i` is defined carefully.

## UDAM claim 1: Repeated observation is a sequential decision

UDAM statement:

```text
first observation and later observations are different decisions
```

Related UDAM expression:

```text
MOV_i > 0
```

Literature fit:

```text
direct support
```

Reason:

Sequential analysis explicitly treats sampling as a process in which data are evaluated as they arrive. Later observations are not merely copies of the first observation. They occur after a changed information state.

This directly supports UDAM's distinction between:

```text
first valid re-anchor observation
```

and:

```text
repeated checking after some evidence has already been obtained
```

Recommended wording:

```text
Sequential analysis directly supports treating repeated observation as a sequence of continue-or-stop decisions rather than as one undifferentiated act of checking.
```

Avoid:

```text
If one observation was useful, repeated observation remains useful.
```

## UDAM claim 2: Continue only while marginal value remains positive

UDAM statement:

```text
MOV_i > 0
```

means the next repeated observation remains favorable.

Literature fit:

```text
partial support
```

Reason:

Optimal stopping supports the general comparison between stopping now and continuing. This is close to UDAM's `MOV_i > 0` rule.

However, `MOV_i` is UDAM-specific notation. It should not be presented as a standard sequential-analysis formula.

There is also a technical caveat:

```text
one-step marginal value != globally optimal continuation value
```

If `MOV_i` means only the immediate expected value of the next observation, then it is a local or myopic rule. If it includes the value of future continuation policies, it is closer to a full optimal-stopping value comparison.

Recommended wording:

```text
Sequential analysis and optimal stopping support the principle that continued observation must be justified by expected continuation value. UDAM expresses this principle through the model-specific MOV_i criterion.
```

Avoid:

```text
MOV_i is Wald's stopping rule.
```

Avoid:

```text
MOV_i > 0 is globally optimal in all sequential problems.
```

## UDAM claim 3: Repeated checking can become harmful or invalid

UDAM statement:

```text
MOV_i <= 0
```

means repeated observation is no longer favorable.

Literature fit:

```text
direct support for the need for stopping rules
partial support for the exact UDAM criterion
```

Reason:

Sequential analysis shows that repeated data looks require explicit stopping rules and error control. Repeated observation is not automatically neutral. It can add cost, delay action, distort inference if handled ad hoc, or produce overchecking loops.

This supports UDAM's caution that repeated checking is different from valid re-anchoring.

Recommended wording:

```text
Repeated observation should stop when its expected continuation value no longer exceeds its cost or when the stopping rule has been met.
```

Avoid:

```text
More checking is always safer.
```

## UDAM claim 4: Stopping requires a defined target and decision rule

UDAM statement:

```text
observation must be tied to a state, a possible action change, and a value condition
```

Literature fit:

```text
direct support
```

Reason:

Classical sequential tests define hypotheses, error thresholds, likelihood-ratio boundaries, and stopping criteria before or during the sequential procedure. Optimal stopping defines a reward or loss criterion.

This supports UDAM's failure-boundary warning:

```text
checking without a state, target, action condition, or cost boundary can become an unbounded loop
```

Recommended wording:

```text
A repeated-observation protocol needs a target state, an update rule, an action condition, and a stopping condition.
```

Avoid:

```text
Observe until it feels certain.
```

## UDAM claim 5: Stop versus switch observation mode

UDAM clarification:

```text
MOV_i <= 0 does not always mean stop all observation
```

Literature fit:

```text
partial support
```

Reason:

Sequential design and adaptive sampling allow later observations to differ from earlier observations. Therefore, low value for the same repeated observation can mean:

```text
stop this observation type
```

not necessarily:

```text
stop all information gathering
```

This is important for UDAM because a checking loop can fail locally even while a different observation would still be valuable.

Recommended wording:

```text
When MOV_i <= 0, the current repeated-observation mode is no longer favorable. A different observation may still be justified if it has positive expected value and can change action.
```

Avoid:

```text
Once repeated checking fails, no further observation is useful.
```

## UDAM claim 6: Sequential analysis proves UDAM

UDAM statement:

```text
not claimed
```

Literature fit:

```text
not supported
```

Reason:

Sequential analysis supports the continue-or-stop layer. It does not provide UDAM's timer seed model:

```text
tau = K + U + R
```

It also does not directly prove:

```text
anchor loss does not imply future anchor invalidation
```

Correct positioning:

```text
Sequential analysis supports UDAM's repeated-observation and stopping-rule discipline, while the anchor-loss synthesis remains UDAM-specific.
```

Avoid:

```text
UDAM is proven by Wald's sequential analysis.
```

## Verification table

| UDAM claim | Sequential-analysis support level | Reason | Correction |
|---|---:|---|---|
| first observation and later observations are different decisions | direct support | sequential analysis evaluates data as they arrive | keep |
| repeated observation needs a stopping rule | direct support | continue/stop is central to the literature | keep |
| `MOV_i > 0` justifies continued observation | partial support | aligned with continuation value, but UDAM-specific notation | do not call it a standard formula |
| `MOV_i <= 0` means checking should stop | partial support | valid for the current observation mode; a different observation may still be useful | clarify stop-this-mode vs stop-all-observation |
| repeated checking can distort inference or waste cost | direct support | sequential methods require stopping/error control | keep |
| stopping requires target, rule, and cost/error boundary | direct support | sequential tests and optimal stopping require explicit criteria | keep |
| timer-derived anchor-loss structure | analogy only | sequential analysis does not contain `tau = K + U + R` | keep UDAM-specific |
| literature proves UDAM as a whole | not supported | only the repeated-observation layer is supported | reject |

## Recommended repository wording

Use:

```text
Sequential analysis supports UDAM's treatment of repeated observation as a continue-or-stop problem: later checks must be justified by their expected continuation value, not by the fact that the first observation was useful.
```

Use:

```text
MOV_i is UDAM-specific notation for marginal observation value. It is aligned with optimal-stopping reasoning but should not be presented as a standard sequential-analysis formula.
```

Use:

```text
When MOV_i <= 0, the current observation mode should stop or be changed; this does not imply that every possible future observation is worthless.
```

Avoid:

```text
MOV_i is Wald's rule.
```

Avoid:

```text
More checking is always safer.
```

Avoid:

```text
Sequential analysis proves UDAM.
```

## Impact on current theory

No major correction is required.

Sequential analysis strengthens the existing UDAM wording around:

```text
first observation versus repeated observation
marginal observation value
checking loops
stopping rules
observation cost
```

One wording correction is recommended:

```text
MOV_i <= 0 should be read as stop or change the current observation mode, not as a global ban on all possible observation.
```

The main caution remains:

```text
component support != full synthesis proof
```
