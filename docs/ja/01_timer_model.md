# タイマーモデル

UDAM の出発点は、タイマーを数える問題です。

## 1. 問題

60秒を数えているとします。

最初はちゃんと数えていた。

しかし途中で気が散り、どれくらい時間がたったかわからなくなった。

そのあと、また数え始めた。

このとき、よくある誤りはこうです。

```text
途中で見失ったから、もう全部意味がない。
```

UDAM はこれを否定します。

## 2. 分解

現在の経過時間をこう分けます。

```text
τ = K + U + R
```

意味：

```text
K = 最初にちゃんと数えた区間
U = 見失った区間
R = 数え直した区間
τ = 現在の経過時間
```

`U` は不確実です。

しかし `R` は新しく観測した区間なので、意味があります。

## 3. 重要な区別

間違った読み方：

```text
U が不明 → 全体が無効
```

正しい読み方：

```text
U が不明 → U は不確実
R は新しいアンカー
```

つまり、失敗したのは全体ではありません。

不明なのは、見失った区間です。

## 4. 絶対不確実性と相対的影響

数え直しても、`U` の不確実性が消えるとは限りません。

```text
Var(U)
```

は残るかもしれません。

しかし、`R` が増えると、全体の中で `U` が占める影響は小さくなる場合があります。

代表的な比率：

```text
ρ = sqrt(Var(U)) / E[τ]
```

意味：

```text
不確実な部分の大きさ / 全体の参照スケール
```

だから、絶対的な不確実性が残っても、相対的な影響は下がることがあります。

## 5. 固定上限

タイマーには、固定された上限があります。

たとえば：

```text
T = 60秒
```

まだ上限に到達していないなら：

```text
K + U + R < T
```

したがって：

```text
U < T - K - R
```

これは、数え直した `R` が増えるほど、`U` の取りうる範囲が狭くなることを意味します。

## 6. まとめ

タイマーモデルの結論はこれです。

```text
見失った区間は不確実になる。
しかし、その後の観測は無効ではない。
```

これが UDAM の最小形です。
