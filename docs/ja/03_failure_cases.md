# 失敗条件と誤用

UDAM は、何でも行動すればよいというモデルではありません。

使える条件と使えない条件があります。

## 1. 観測が状態と関係ない

失敗条件：

```text
P(y | S) = P(y)
```

意味：

```text
観測 y が、状態 S について何も教えていない。
```

この場合、その行動は再アンカーではありません。

例：

```text
勉強状態を知りたいのに、関係ない動画を見る
作業状態を知りたいのに、ただ机を片付け続ける
```

## 2. 観測しても次の行動が変わらない

観測の価値は、次の行動を変えられることにあります。

失敗条件：

```text
a(y) = a_0
```

意味：

```text
何を見ても、結局同じ行動をする。
```

この場合、観測の価値は低くなります。

## 3. 観測コストが高すぎる

観測にはコストがあります。

```text
C(obs)
```

観測による改善よりもコストが大きいなら、観測は不利です。

```text
OV <= 0
```

## 4. 確認ループになる

反復観測は、価値がある間だけ有効です。

条件：

```text
MOV_i > 0
```

失敗条件：

```text
MOV_i <= 0
```

意味：

```text
もう次の確認で行動が変わらないのに、確認だけ続ける。
```

これは UDAM ではなく、空回りです。

## 5. 広げすぎる

小さく試してうまくいったあと、広げることはあります。

しかし、広げすぎると失敗します。

失敗条件：

```text
B_expand(r_i) + I_expand(r_i)
<=
C_obs(r_i) + P_boundary(i) * C_boundary + C_correct(r_i)
```

意味：

```text
広げる利益より、観測コスト・境界リスク・修正コストの方が大きい。
```

## 6. 境界リスクを低く見積もる

特に危ないのは、次の観測前に不利な境界を越えることです。

```text
P_boundary(i) * C_boundary
```

これを小さく見積もりすぎると、広げ方を間違えます。

## 7. 不可逆なのに大きく広げる

行動が戻せない場合、拡張倍率は小さくするべきです。

失敗例：

```text
一度やると戻せない行動を、十分な観測なしに大きく広げる
```

UDAM では、可逆性が低いほど慎重にします。

## 8. 局所観測から全体結論を出す

UDAM は局所観測を重視します。

しかし、局所観測だけで全体を断定してはいけません。

失敗例：

```text
1問解けなかった -> 全部だめ
1回返事が悪かった -> 関係は終わり
1つ作業が詰まった -> プロジェクト全体が無理
```

正しくは：

```text
局所観測は、局所状態を更新する。
全体結論には追加観測が必要。
```

## 9. まとめ

UDAM が失敗しやすいのは、次のときです。

```text
状態と関係ない観測
次行動が変わらない観測
高すぎる観測コスト
確認ループ
広げすぎ
境界リスクの過小評価
不可逆行動の過大拡張
局所観測からの全体断定
```

短く：

```text
小さく見るだけでは足りない。
状態と関係があり、行動を変え、コストに見合う観測でなければならない。
```
