# 実践プロトコル

UDAM を実際に使うときの手順です。

## 1. 全体を判定しない

不確実なとき、人はすぐに全体結論を出しがちです。

```text
もう無理
全部だめ
終わった
意味がない
```

UDAM では、まずこの全体判定を止めます。

代わりに、局所状態を観測します。

```text
全体を決める前に、今の一部を確かめる。
```

## 2. 状態を決める

まず、何について不確実なのかを決めます。

例：

```text
学習状態
作業状態
人間関係の状態
体調の状態
生活の選択肢
```

これを `S` と考えます。

```text
S = 今の隠れた状態
```

## 3. 小さく観測する

次に、小さい観測を1つ選びます。

条件はこれです。

```text
その観測が、状態 S と関係していること。
```

形式的には：

```text
P(y | S) != P(y)
```

意味：

```text
観測結果 y が、状態 S について何かを教えてくれる。
```

## 4. 結果で次の行動を変える

観測の価値は、結果によって次の行動が変わることにあります。

```text
観測結果 -> 次の行動
```

例：

```text
1問解けた -> 次に2問やる
1問も解けない -> 解説を見る
作業リストが小さい -> 1つ着手する
作業リストが大きい -> まず分解する
```

## 5. 観測し続けない

1回目の観測は価値が高くなりやすいです。

しかし、2回目以降は別です。

次の問いを使います。

```text
次の観測で、次の行動が変わるか？
```

変わるなら見る。

変わらないなら、行動するか、いったん止める。

## 6. うまくいったら広げる

小さな観測がよい結果を返したら、範囲を広げてもよいです。

```text
s_{i+1} = r_i s_i
```

例：

```text
1 -> 2 -> 4 -> 8
```

ただし、いつも2倍がよいわけではありません。

## 7. 境界リスクを見る

広げすぎると、次に観測する前に不利な境界を越えることがあります。

そのため、拡張は次の条件で考えます。

```text
B_expand(r_i) + I_expand(r_i)
>
C_obs(r_i) + P_boundary(i) * C_boundary + C_correct(r_i)
```

直感的には：

```text
広げる利益と情報価値
>
観測コスト + 境界リスク + 修正コスト
```

## 8. まとめ

```text
1. 全体を判定しない
2. 状態を決める
3. 小さく観測する
4. 結果で次の行動を変える
5. 観測し続けない
6. うまくいったら広げる
7. 境界リスクが高いなら広げすぎない
```

短く：

```text
小さく観測し、結果で行動を変え、リスクで広げ方を制約する。
```
