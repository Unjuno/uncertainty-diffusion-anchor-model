# 位置づけ

UDAM は、完全に新しい数学理論ではありません。

より正確には、既存の考え方をタイマー問題から整理した、実践的な統合モデルです。

## 1. 新しくない部分

UDAM には、既存の考え方に近い部分が多くあります。

例：

```text
小さく試す
反応を見る
更新する
うまくいけば広げる
危なければ止める
```

これは、一般的には次のような考え方に近いです。

```text
試行錯誤
フィードバック
ベイズ更新
情報価値
探索と活用
オンライン意思決定
```

だから、UDAM を次のように言うのは過剰です。

```text
完全に新しい不確実性理論
```

## 2. UDAM の独自な整理

UDAM の中心は、タイマー問題から来ています。

```text
τ = K + U + R
```

意味：

```text
K = わかっている部分
U = 見失った部分
R = 再開して新しくわかった部分
```

ここから出る中心命題は：

```text
見失った部分があることは、これからの観測が無効になることを意味しない。
```

英語では：

```text
Anchor loss does not imply future anchor invalidation.
```

## 3. 一般論との差分

一般論：

```text
小さく試して、フィードバックを見て、調整する。
```

UDAM：

```text
アンカー喪失によって全体無効化の誤りが起きる。
しかし K + U + R に分けると、失われたのは U の確実性であり、R の有効性ではない。
```

この分解が、UDAM の説明力です。

## 4. 趣味プロジェクトとしての位置づけ

これは、論文として完成させることが目的ではありません。

目的は：

```text
自分の発想を、破綻しにくい形で整理すること。
```

そのためには、以下が重要です。

```text
説明しやすいこと
反例があること
誤用を防ぐこと
既存理論と矛盾しにくいこと
```

## 5. 推奨する説明

```text
UDAM は、アンカー喪失後の再アンカーを扱う、タイマー由来の実践的統合モデルである。
```

さらに短く：

```text
部分的な不確実性は、全体の無効化を意味しない。
```

## 6. 避ける説明

```text
UDAM は完全に新しい数学理論である。
```

```text
UDAM はどんな状況でも行動を肯定する。
```

```text
UDAM は常に2倍で広げることを推奨する。
```

これらはすべて誤読です。

## 7. 最終的な位置づけ

```text
既存理論と整合する、タイマー由来の実践的統合モデル。
```
