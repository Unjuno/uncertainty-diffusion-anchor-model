# 平易な説明

## 1. 何の話か

UDAM は、何かを途中で見失ったときに、そこからどう立て直すかを整理するモデルです。

出発点はタイマーです。

60秒を数えている。最初の20秒はちゃんと数えた。しかし途中で気が散って、どれくらい時間がたったかわからなくなった。そのあと、また10秒数えた。

このとき、よくある誤りはこうです。

```text
途中で見失ったから、もう全部意味がない。
```

UDAM では、そうは考えません。

```text
見失った部分は不確実。
しかし、そのあと新しく数えた部分は有効な情報。
```

## 2. 最小式

UDAM の最小式はこれです。

```text
τ = K + U + R
```

意味：

```text
K = アンカー喪失前にわかっている部分
U = 見失った不明な部分
R = 再アンカー後に新しくわかった部分
τ = 現在位置
```

重要なのは、`U` が不明でも `R` は無効にならないことです。

## 3. 中心命題

```text
過去のアンカー喪失は、未来の有効観測を無効化しない。
```

別の言い方をすると：

```text
部分的な不確実性は、全体の無効化を意味しない。
```

これは、学習・仕事・人間関係・生活の立て直しにも使えます。

## 4. 使い方

まず、全体を判定しようとしない。

代わりに、局所的に観測します。

```text
今の状態を少しだけ確かめる。
```

例：

```text
1問だけ解く
作業リストを開く
1つだけ記録する
低圧の連絡を1回する
```

その結果で次の行動を変えます。

```text
観測結果 -> 次の行動
```

## 5. 何でも行動すればよいわけではない

UDAM は、行動一般を肯定していません。

有効なのは、状態について情報を返す行動です。

条件：

```text
P(y | S) != P(y)
```

意味：

```text
観測 y が、状態 S と関係している。
```

状態と関係ない行動は、再アンカーではありません。

## 6. 1回目と2回目以降は違う

観測ゼロの状態では、最初の観測の価値は高くなりやすいです。

しかし、2回目以降は別です。

判断基準は：

```text
次の観測で、次の行動が変わるか？
```

変わるなら観測する価値があります。

変わらないなら、観測を続けるより行動した方がよい場合があります。

## 7. うまくいったら広げる

小さく試して、反応がよければ広げることがあります。

```text
1 -> 2 -> 4 -> 8
```

これは便利な広げ方ですが、常に正しいわけではありません。

広げすぎると、次に観測する前に不利な境界を越える場合があります。

だから、広げ方は次の条件で制約されます。

```text
B_expand(r_i) + I_expand(r_i)
>
C_obs(r_i) + P_boundary(i) * C_boundary + C_correct(r_i)
```

直感的には：

```text
広げる利益と情報価値
>
観測コスト + 境界リスク + 修正コスト
```

## 8. まとめ

```text
小さく観測する。
結果で次の行動を変える。
反応がよければ広げる。
ただし、境界リスクが高いなら広げすぎない。
```

さらに短く：

```text
見失っても、そこから先の有効観測は残る。
```
