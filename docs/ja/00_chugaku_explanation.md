# 中学生向け説明

## 1. 何の話か

これは、何かを途中で見失ったときに、どうやってまた立て直すかの話です。

たとえば、60秒を数えていたとします。

最初の20秒はちゃんと数えた。

でも、そのあと気が散って、何秒たったかわからなくなった。

そのあと、また10秒数え始めた。

このとき、普通はこう思いやすいです。

```text
途中で失敗したから、もう全部意味ない。
```

でも、UDAM ではこう考えます。

```text
途中でわからなくなった部分はある。
でも、そのあと数えた10秒は意味がある。
```

## 2. 式で見る

UDAM の一番大事な式はこれです。

```text
τ = K + U + R
```

意味はこうです。

```text
K = ちゃんとわかっている最初の部分
U = わからなくなった部分
R = もう一度始めてからわかった部分
τ = 今どこにいるか
```

たとえば：

```text
K = 20秒
U = わからない
R = 10秒
```

なら：

```text
τ = 20秒 + わからない部分 + 10秒
```

ここで大事なのは、`U` がわからないからといって、`R` まで無意味になるわけではないことです。

## 3. いちばん大事な考え方

```text
わからない部分があることと、
これから確かめることが意味ないことは別。
```

これが UDAM の中心です。

もっと短く言うと：

```text
途中で見失っても、そこから先はまたアンカーにできる。
```

## 4. 学習にたとえる

テスト勉強をしていたとします。

でも、途中で何日もサボってしまった。

そのとき、こう思うかもしれません。

```text
もう全部忘れた。
だからやっても無駄。
```

でも、本当はまだ確かめていません。

だから、まず1問だけ解いてみる。

その1問は、新しいアンカーになります。

結果がよければ：

```text
思ったより覚えているかもしれない
```

結果が悪ければ：

```text
どこが抜けているかわかった
```

どちらにしても、今の状態が少し見えます。

## 5. 行動する前に必要な条件

UDAM は「何でも行動しろ」とは言っていません。

大事なのは、その行動が状態を教えてくれるかどうかです。

良い行動：

```text
1問解く
作業リストを1回見る
小さく確認する
1つだけ記録する
```

悪い行動：

```text
意味のない確認を何度もする
ただ焦って動く
状態と関係ないことをする
```

## 6. 1回見たあとはどうするか

最初に1回見るのは大事です。

でも、見続ければいいわけではありません。

次に見るかどうかは、こう考えます。

```text
次の観測で、次の行動が変わるか？
```

変わるなら見る価値があります。

変わらないなら、見るより行動した方がいい場合があります。

## 7. うまくいったら広げる

小さく試してうまくいったら、少し広げてもよいです。

たとえば：

```text
1問できた
→ 次は2問やる
2問できた
→ 次は4問やる
```

これは、少しずつ広げる考え方です。

でも、いつも2倍にすればよいわけではありません。

危ないときは、広げすぎない方がいいです。

## 8. 危ない境界

次に確かめる前に、悪いことが起きる可能性があるなら、広げすぎてはいけません。

UDAM ではこれを、抽象的にこう考えます。

```text
次の観測前に、不利な境界を越えるリスク
```

たとえば：

```text
広げすぎる
→ 次に見る前に手遅れになる
```

ような場合です。

だから、広げ方はリスクで調整します。

## 9. 一言でまとめる

```text
小さく確かめる。
結果で次の行動を変える。
うまくいけば広げる。
危ないなら広げすぎない。
```

もっと短く：

```text
見失っても、そこからまた確かめればいい。
```
