# 07 Applications

UDAM can be applied wherever an agent loses an anchor and uncertainty about the current state grows over time.

Each application should be written in the same structure:

1. Situation
2. Lost anchor
3. Uncertainty diffusion
4. Re-anchoring action
5. Informational value
6. Intervention value
7. Failure case

## Application domains

| Domain | Lost anchor | Re-anchor |
|---|---|---|
| time perception | elapsed time | restart counting |
| learning | skill calibration | solve one problem |
| work | project state | inspect task list or working tree |
| relationships | social state | send one clear message |
| health | body state | record one variable |
| life strategy | self-state after failure | take one small reality-contacting action |

## Practical principle

The first action after anchor loss should usually be small, concrete, and diagnostic.

It should answer:

> What state am I actually in now?

## Bad application pattern

Do not use UDAM to justify:

- blind optimism;
- compulsive action;
- vague productivity;
- high-risk action under uncertainty;
- ignoring safety or professional support.

## Good application pattern

Use UDAM to select actions that:

- reduce uncertainty;
- reveal the next action;
- create feedback;
- improve the state slightly;
- preserve future option value.
