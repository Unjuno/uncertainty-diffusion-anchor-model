# Concrete Example: Relationship Re-Anchor

This example shows how UDAM can be used when a social state is unclear.

## Situation

Someone has not replied for a while.

The agent begins imagining several possible meanings.

## Lost anchor

The current social state is unclear.

Possible states:

```text
busy
neutral
unclear
less engaged
```

## Common mistake

The agent jumps from uncertainty to a global conclusion.

```text
I do not know what this means -> I assume the whole relationship state
```

## Re-anchor action

Send one low-pressure, specific message.

Example:

```text
No rush. Just checking whether this week still works for you.
```

## If you are in this situation, do this now

```text
1. Write one low-pressure, specific message.
2. Send it once.
3. Classify the response pattern: normal / busy / unclear / no reply.
```

Do not use the message to force the other person to decide.

Use it to make the social state more observable.

## Observation result

The result separates possible states:

```text
normal reply
busy reply
unclear reply
no reply
```

Each result changes the next action.

## Next action map

```text
normal reply -> continue normally
busy reply -> wait or reschedule
unclear reply -> ask one clarifying question later
no reply -> stop repeated checking and update the state cautiously
```

## UDAM interpretation

The message is not meant to force an outcome.

It is a low-cost observation that updates the social state.

```text
OV > 0
```

## Best practice

```text
send one clear low-pressure signal -> update from the response pattern
```

## Boundary

Do not send repeated messages if one signal already gives enough information for the next decision.

Repeated observation is useful only while:

```text
MOV_i > 0
```
