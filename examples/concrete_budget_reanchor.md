# Concrete Example: Budget Re-Anchor

This example shows how UDAM can be used when a personal budget state is unclear.

## Situation

A person avoids looking at their remaining budget.

They do not know whether the current month is comfortable or tight.

## Lost anchor

The budget state is unclear.

Possible states:

```text
comfortable
tight
needs adjustment
```

## Common mistake

The person treats the unknown number as something to avoid.

```text
I do not know the number -> I avoid looking -> the state stays unclear
```

## Re-anchor action

Write down three numbers:

```text
current amount
next planned expense
remaining days
```

## Observation result

The result separates possible states:

```text
comfortable
tight but adjustable
needs a small plan
```

Each state suggests a different next action.

## UDAM interpretation

An observed constraint is more actionable than vague pressure.

The action has observability value when it changes the next decision.

```text
OV > 0
```

## Best practice

```text
extract the minimum numbers needed for the next decision
```

## Boundary

Do not keep re-checking the same numbers if the next action is already clear.

Repeated checking is useful only while:

```text
MOV_i > 0
```
