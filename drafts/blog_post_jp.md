# 不確実性拡散アンカーモデル：小さな情報行動が合理的になる理由

## 導入

一度ミスったからといって、その後の計測や行動がすべて無意味になるわけではない。

たとえば、ある固定された目標条件に向かって時間を数えていて、途中で見失ったとする。失われた区間は不確実になる。しかし、そこから再び数え始めれば、再開後のカウントは新しい情報になる。

この直感を一般化したものが、不確実性拡散アンカーモデル、UDAM である。

## 何を扱うモデルか

UDAM が扱うのは、主に次の状況である。

```text
fixed target, uncertain position
```

つまり、対象条件は固定されているが、自分が今どこにいるかが分からない状況である。

これは、次とは違う。

```text
uncertain target, uncertain position
```

対象そのものが起きるかどうかまで不確実な場合は、別のレイヤーが必要になる。

UDAM の中核では、その拡張は扱わない。理由は単純で、対象の発生そのものが非制御可能なら、再アンカーの価値が曖昧になるからである。

## 基本式

タイマー種モデルは次の式で表せる。

```text
τ = K + U + R
```

- `K`: 見失う前に分かっている区間
- `U`: 見失った不明区間
- `R`: 数え直した後の区間
- `τ`: 現在位置

不明なのは `U` であり、`R` まで無意味になるわけではない。

つまり、よくある誤りはこうである。

```text
一部が分からない → 全部無意味
```

UDAM の見方はこうである。

```text
一部が分からない → その部分だけ不確実 → 次のアンカーは作れる
```

## 絶対不確実性と相対的影響

再アンカーは、失われた区間 `U` の不確実性を必ず下げるわけではない。

むしろ、厳密には `U` の不確実性が増える場合もある。

```text
Var(U_{t+Δt}) >= Var(U_t)
```

しかし、それでも `U` の相対的な影響は小さくなる場合がある。

代表的には、次の比で考えられる。

```text
ρ = sqrt(Var(U)) / E[τ]
```

全体の参照スケール `E[τ]` が、不確実性スケール `sqrt(Var(U))` より速く大きくなるなら、相対的影響は下がる。

つまり、重要なのはこうである。

```text
絶対不確実性は増える場合がある
しかし相対的影響は小さくなる場合がある
```

## 不確実性は放置で増えうる

アンカーがない状態では、現在状態についての不確実性は時間とともに増えうる。

```text
P_{t+Δt} = P_t + QΔt
```

ただし、これは「世界が必ず悪くなる」という意味ではない。

UDAM が主に扱うのは、外部状態そのものではなく、状態についての信念不確実性である。

```text
state dynamics: S_t changes
belief dynamics: P_t changes
```

状態が変わることと、状態について分からなくなることは別である。

## 行動の価値

行動の価値は次のように考える。

```text
V(a) = I(a) + B(a) - C(a)
```

- `I(a)`: 情報価値
- `B(a)`: 介入価値
- `C(a)`: 行動コスト

`V(a) > 0` なら、その行動には打つ価値がある。

重要なのは、UDAM が「何でも行動しろ」と言っているわけではないということだ。

情報を返さない行動、状態を改善しない行動、コストが高すぎる行動は、再アンカーではなくノイズである。

## 制御可能性の境界

UDAM は行動志向のモデルである。

したがって、中核に入れる不確実性は、行動によって何らかの形で扱える必要がある。

たとえば、行動によって次のどれかが可能である場合である。

- 観測できる
- 信念を更新できる
- 意思決定を改善できる
- 状態そのものに介入できる

逆に、対象の発生そのものが非制御可能なら、それは中核モデルから外すべきである。

それは外部条件、例外条件、または別の拡張として扱う。

## 適用できるのに適用しない誤算

UDAM で説明できる重要な失敗は、理論の失敗ではなく、エージェント側の誤算である。

典型例はこれである。

```text
partial uncertainty → total invalidation
```

一部が不確実になっただけなのに、全体を無効だと思って止まってしまう。

たとえば、

- 一部の計測を逃したので全部無意味だと思う
- 勉強を中断したので再開しても意味がないと思う
- プロジェクト状態が不明なので開くことすら避ける
- 関係性が不明なので低圧な一通も送らない
- 確信が出るまで診断的行動をしない

しかし、UDAM の修正はこうである。

```text
partial uncertainty → re-anchor if the next action has positive value
```

## 高緊張な例の扱い

高緊張な例は、固定ターゲット、位置不確実性、アンカー喪失、再アンカーの構造を見せるには有効である。

ただし、公開文書では抽象化する。

保存すべきなのは、危険な細部ではなく、次の構造である。

```text
fixed target
uncertain position
lost anchor
re-anchoring action
```

そのため、UDAM では高緊張な例を使う場合も、中立的・非操作的な表現にする。

## 結論

不確実性が拡散する状況では、大きな一発逆転より、小さな情報行動を細かく打つことが合理的になりうる。

ただし、それは「とにかく動け」という意味ではない。

正確にはこうである。

> 固定された対象に対して現在位置が不確実なとき、低コストで情報を返す行動は、新しいアンカーを作るため合理的になりうる。

そして、最も避けるべき誤算はこれである。

```text
一部が分からないから、全部無意味
```

UDAM はこう言う。

```text
一部が分からないなら、そこを不確実なまま置く。そして次のアンカーを作る。
```
